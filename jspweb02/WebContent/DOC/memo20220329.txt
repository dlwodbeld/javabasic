데이터 전달 방식 2가지(default GET)
GET		주소창으로 전달			보안에 취약		대용량 불가		속도가 빠르다.			
POST	파일내부로 심어서 전달		보안에 우수		대용량 가능		속도가 상대적으로 느리다.	

GET방식은 주소창에 사용자가 입력한 값들이 그대로 노출되는 것을 볼 수 있다.
패스워드 같은 보안이 필요한 값들까지 노출이 되면 문제가 된다.
Post방식을 사용하면 이러한 문제점을 해결할 수 있다.

servlet 서블렛은 Sever Applet 의 약자 자바기반의 동적 웹 프로그래밍 솔루션이다.
서블렛을 기반으로 JSP가 개발되었다.따라서 서블렛의 구성,동작방식을 이해하면 JSP를 이해하는데도 많은 도움이 된다.

JSP java servlet page JSP는 서블렛에 비해 쉬운 프로그래밍방법을 제공한다. 하지만 내부적으로는 서블렛으로 변환하여 실행한다.

servlet은 자바코드 사이에 HTML코드를 끼워넣어서 프로그래밍을 하는 형태이고 반대로 JSP는 HTML 코드 사이에 자바 코드를 끼워 넣어서
프로그래밍을 하는 형태이다.


JSP

JSP 스크립팅 요소 5가지

스크립틀릿(Scriptlet)	<%	%> 자바코드영역
주석(comment) 		<%-- --%> 주석처리(모두 사용 가능)
지시어(Directive)		<%@		%> 정보
선언문(Declaration)	<%!		%> 함수 및 변수 정의
표현식(Expression)	<%=		%>함수 변수 값 호출

1 스크립틀릿(Scriptley)

가장 기본이 되는 스크립틀릿은 Java코드를 삽입하기 위한 목적으로 사용된다.
스크립틀릿 안에 정의되 코드는 서블릿으로 변환할 때 그대로 들어가게 된다.

2주석
HTML 주석의 형태와 매우 유사하다.

<!== html주석 -->
<%-- jsp주석 --%>


3. 지시어(Directive)
<%@		%> 			@는 애너테이션 즉 주석이나 부가 정보를 의미한다.
<%@	page ...%> 		페이지 처리 방법 정의
<%@	include ...	%>	다른 페이지 삽입(import)
<%@	taglib ...%> 	태그라이브러리 접근

4.선언문(Declaration)

<%!		%>

함수나 변수의 선언이 가능하다.
대부분의 경우 스크립틀릿에서 선언이 가능함에따라 사용빈도가 적다.
하지만 반드시 선언문에서 선언을해야할 경우도있다.

5.표현식(Expression)
<%=		%>
표현식 태그로 함수나 변수의 호출이 가능하다.

표현식의 코드가 변수일 경우에는 실행시 변수에 저장된 값이 출력되고 함수일 경우에는
주어진 함수를 호출하여 얻은 반환값이 출력된다.
